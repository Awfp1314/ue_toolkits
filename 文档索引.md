## 📝 文档维护

### 更新文档

当代码变更时，记得同步更新文档：

1. 添加新功能 → 更新相应的 README
2. 修改 API → 更新 API 参考章节
3. 添加示例 → 添加到使用示例中
4. 修复 bug → 更新故障排查章节

### 文档风格

- 使用 Markdown 格式
- 包含代码示例
- 提供实际用例
- 添加表格和图表
- 使用 emoji 增强可读性 ✨
